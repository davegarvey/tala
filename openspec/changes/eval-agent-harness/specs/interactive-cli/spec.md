## ADDED Requirements

### Requirement: Interactive mode guides the user step by step

When run without `--auto`, the harness SHALL operate in interactive mode. After each `advance` action, the harness SHALL print:
- What just happened (daemon started, feedback collected, etc.)
- The current state
- The recommended next action(s) with explicit instructions

The harness SHALL NOT proceed to the next state without an explicit `advance` call.

#### Scenario: After setup, harness prints next steps
- **WHEN** the user calls `./eval/harness.sh advance setup` from `initial`
- **THEN** the harness runs setup, prints daemon PID, prints `Next: Launch sub-agents. Copy prompts from:` followed by the task prompt file paths, and prints `After sub-agents complete, run: ./eval/harness.sh advance collecting`

#### Scenario: After collect, harness prints critique instructions
- **WHEN** the user calls `./eval/harness.sh advance collecting`
- **THEN** the harness stops the daemon, aggregates feedback, prints a summary, and prints `Next: ./eval/harness.sh advance critiquing`

### Requirement: Auto mode outputs structured KEY=VALUE with file paths

When run with `--auto`, the harness SHALL output `KEY=VALUE` pairs, one per line. Multi-line content SHALL NOT be embedded inline — the harness SHALL write content to a file and output `KEY_FILE=<path>` instead.

#### Scenario: Auto status outputs key=value pairs
- **WHEN** `./eval/harness.sh --auto status` is called
- **THEN** output SHALL include `STATE=launching LOOP=1 SCENARIO=cross-project AVAILABLE=collecting` on separate lines

#### Scenario: Auto advance outputs file paths not inline content
- **WHEN** `./eval/harness.sh --auto advance setup` completes
- **THEN** output SHALL include `TASK_PROMPT_ALPHA_FILE=eval/agent-tasks/cross-project/agent-alpha.md` and never inline the prompt content

### Requirement: Save-critic subcommand

The harness SHALL provide a `save-critic` subcommand that reads JSON from stdin, validates the structure (must contain `p0`, `p1`, `p2` arrays), and writes it to `eval/agent-tasks/<scenario>/critic-output-loop-<N>.json` where N is the current loop count.

#### Scenario: Valid JSON is saved
- **WHEN** the user runs `echo '{"p0":[],"p1":[],"p2":[]}' | ./eval/harness.sh save-critic`
- **THEN** the harness validates the structure, writes to `eval/agent-tasks/<scenario>/critic-output-loop-1.json`, and prints `Critic output saved to ...`

#### Scenario: Invalid JSON is rejected
- **WHEN** the user runs `echo 'not json' | ./eval/harness.sh save-critic`
- **THEN** the harness prints a parse error, does NOT write any file, and exits with code 1

#### Scenario: Missing required keys is rejected
- **WHEN** the user runs `echo '{"p0":[]}' | ./eval/harness.sh save-critic`
- **THEN** the harness prints `Missing required key: p1` and exits with code 1

### Requirement: Task prompt file generation

The harness SHALL write agent task prompt files to `eval/agent-tasks/<scenario>/` during the `setup` transition. Each file SHALL contain the full agent task definition as generated by the scenario's setup function.

#### Scenario: Task prompt files exist after setup
- **WHEN** the harness completes the `advance setup` transition
- **THEN** `eval/agent-tasks/<scenario>/` SHALL contain one `.md` file per agent role

### Requirement: Critic prompt generation with structured output requirement

The harness SHALL generate the critic prompt during the `critiquing` transition. The critic prompt SHALL instruct the critic sub-agent to output structured JSON in the following format (returned inline, not written to file):

```json
{
  "p0": [{"description": "...", "rationale": "..."}],
  "p1": [{"description": "...", "rationale": "..."}],
  "p2": [{"description": "...", "rationale": "..."}],
  "summary": "..."
}
```

#### Scenario: Critic prompt includes structured output instructions
- **WHEN** the harness generates the critic prompt
- **THEN** the prompt SHALL include the exact JSON schema above and instruct the critic to return the JSON in its Task result

### Requirement: Deterministic exit decision from structured critic output

The harness SHALL read `eval/agent-tasks/<scenario>/critic-output-loop-<N>.json` during exit criteria evaluation. If `p0` and `p1` arrays are both empty, the harness SHALL treat this as "no material issues found" and offer `advance exit`. If either array has items, the harness SHALL treat this as "issues found" and offer `advance spec`.

#### Scenario: Exit on empty p0 and p1
- **WHEN** `critic-output-loop-1.json` contains `{"p0": [], "p1": [], "p2": ["wish"]}`
- **THEN** the harness SHALL print "No P0 or P1 items. Exit criteria met." and offer `advance exit`

#### Scenario: Continue on p0 or p1 items
- **WHEN** `critic-output-loop-1.json` contains `{"p0": ["crash"], "p1": ["confusing"], "p2": []}`
- **THEN** the harness SHALL print "2 material issues found. Open a spec change to address them." and offer `advance spec`

#### Scenario: Missing critic output file asks user to save
- **WHEN** `critic-output-loop-1.json` does not exist at exit check time
- **THEN** the harness SHALL print "Critic output not found. Use 'save-critic' to save the critic's JSON response, then run 'advance analyzing' again."
